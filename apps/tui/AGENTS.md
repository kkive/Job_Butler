# apps/tui/AGENTS.md

## 目标
提供 ratatui 终端界面，支持任务启动、进度展示和配置管理。

## 页面结构
- 顶部菜单栏：`首页`、`任务`、`设置`
- 首页：
  - 展示 Logo
  - 展示项目介绍
  - 提供 `开始任务` 按钮
- 任务：
  - 展示任务列表/进度（当前阶段可先占位）
- 设置：
  - 查看服务商
  - 添加服务商
  - 删除服务商
  - 反馈问题
  - 配置

## 视觉主题
- 主强调色：`rgb(110, 228, 149)`
- 背景色：黑色
- 文字色：白色
- 高亮态：强调色背景 + 黑色文字
- 弹窗主题色：`#F4D1B4`（用于查看/添加/删除服务商弹窗）

## 交互规范
- 键盘：
  - `Left/Right` 切换菜单
  - `Up/Down` 在设置页移动选择
  - `Enter` 执行当前项
  - `Q` 或 `Esc` 退出
- 鼠标：
  - 支持点击顶部菜单切换页面
  - 支持点击首页开始按钮
  - 支持点击设置列表项触发动作
  - 鼠标左键按下和抬起均视为点击触发

## 启动前置
- `apps/tui` 启动前必须先检查数据库状态
- 若 sqlite 不存在或异常，调用 `crates/storage` 的初始化/恢复流程自动处理
- 仅在数据库就绪后进入 UI 事件循环

## 设置页数据访问
- `查看服务商 / 添加服务商 / 删除服务商` 必须通过 `crates/api` 调用
- `apps/tui` 不直接操作数据库
- 这三项操作必须使用弹窗页面交互
- `添加服务商` 需由用户手动填写表单后提交
- 弹窗显示时使用模态遮罩，弱化后景内容，避免信息重叠干扰
- 弹窗打开时仅保留顶部菜单栏和底部状态栏可见，中间内容区必须先清空再渲染弹窗

## 输入
- 用户键盘/鼠标事件
- orchestrator 推送的任务状态（后续接入）

## 输出
- 启动/停止任务命令（后续接入 orchestrator）
- UI 状态更新

## 约束
- 不包含业务投递逻辑
- 不直接控制鼠标或窗口
